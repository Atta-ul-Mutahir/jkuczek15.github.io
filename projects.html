The MU – Group Project

This is a web / iOS application created to give students one platform where they can view free food items on campus. They are also able to buy/sell textbooks, tickets, furniture and other items. I worked on the web side of the application, which utilized Google’s Firebase service as a backend and Angular 2 as a client-side framework. We chose these technologies because Angular is a very popular framework by Google, and we though Angular would integrate nicely with Firebase on the backend. We chose Firebase as a backend because we wanted real-time updates, and Firebase provided simple guides for reading and writing data on both iOS and web.

Features:

  • Single page web application.
  • User-authentication system utilizing third party authentication (Google) and limiting access to only users with an “@iastate.edu” email.
  • Multiple levels of authentication with different access rights: user, moderator, admin
  • Fully responsive real-time chat that can communicate with the iOS application.
  • Users can view, edit, create, and delete different items.
  • Ability to view your own “for-sale” items, ability to upload attachments, ability to block users.
  • Real-time updates throughout the entire application.

Website for Triangle Fraternity – Group Project

This is a web application created for the Triangle Fraternity at Iowa State. The client needed to be able to update content with upcoming events or announcements. We created a single page web application utilizing MySQL as a database, Spring Framework on the backend, and standard HTML / JavaScript + jQuery / CSS on the frontend. We decided to create this backend because we weren’t familiar with an enterprise server-side framework like Spring. We chose a simple frontend because the website didn’t require a lot of complex/shared components on the client side.

Features:

  • Single page web application.
  • Custom scrolling navigation bar
  • User editable content (links, text, images, and HTML)
  • Secure service oriented multi-layered architecture.
  • Custom basic authentication system utilizing SSL.

Rouge-like Dungeon Crawler Application – Individual

Created a text-based desktop game that includes all the basics of C++ (IO, file streams, pointers, vectors, classes, etc.). Converted the game into an Android application, which also utilizes Java. Developed an understanding of the differences between high and low-level programming languages. Utilized the Android NDK library to run native C++ code on an Android device. As well as developing this project, I came up for the idea. Our professor gave us a lot of freedom when creating the final version of our project, so I decided to create an Android app.

Features:

  • Implementation of Dijkstra’s pathfinding algorithm to find the shortest path to the player.
  • Graphical mobile interface given to traditional desktop dungeon game.
  • Different types of monsters and monster abilities imported from custom configuration file.
  • Player has an inventory, HP, can drop and pickup items that allow them to deal more damage.
  • Players can travel between levels of the dungeon each with a different difficulty.
